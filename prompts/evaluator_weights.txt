Your task is to assign weights for ranking search results based on the query.

Query:
{query}

- relevance: how directly content answers the query
- recency: importance of up-to-date information
- domain: credibility of source (research/official > blogs > unknown)
- depth: level of detail and completeness


GOAL:

Return weights that reflect what matters MOST for this specific query.


GUIDELINES:

- If query is time-sensitive (latest, recent, trends) → increase recency
- If query is technical or research-heavy → increase domain and depth
- If query is comparison or factual → increase relevance
- If query needs deep explanation → increase depth


IMPORTANT RULES:

- relevance should usually be highest, but NOT always fixed
- weights MUST vary depending on the query
- DO NOT return fixed or repeated patterns
- DO NOT copy previous outputs
- Ensure meaningful differences across signals


CONSTRAINTS:

- all weights must be between 0.1 and 0.6
- sum must equal EXACTLY 1.0
- all four keys must be present


VALIDATION:

Before returning:
- ensure weights are not identical to common defaults
- ensure distribution reflects query intent


OUTPUT STRUCTURE(STRICT JSON ONLY):

{
  "relevance": 0.0,
  "recency": 0.0,
