* DO NOT introduce new facts
* DO NOT hallucinate
* DO NOT infer missing information

CONTROLLED TRANSFORMATION RULE

Each claim contains:

* text

* Convert EACH claim into EXACTLY ONE bullet point
* Preserve factual meaning strictly
* Perform ONLY minimal linguistic cleanup for readability
* DO NOT reinterpret or expand meaning

* One claim → one bullet
* Do NOT split claims
* Do NOT merge claims

Bullet format:

QUERY-AWARE STRUCTURING (DETERMINISTIC)

You MUST determine the query intent and structure the report accordingly.

STRICT PRIORITY ORDER:

1. Explicit query intent (pros/cons, comparison, etc.)
2. Research question alignment (multi-step queries)
3. Semantic grouping (fallback only)


STRUCTURING RULES

1. If query asks for pros/cons:
   → Create BOTH sections:
   - Advantages
   - Disadvantages

2. If query is comparative:
   → Use:
   - Entity-based comparison OR
   - Aspect-based comparison

3. If query is informational:

   FIRST classify claims into:

   - Definitions / Concepts
   - Types / Categories
   - Applications / Use Cases
   - Implications / Ethics / Risks

   THEN group into sections accordingly

4. If query is mixed/ambiguous:
   → Use best-effort grouping WITHOUT forcing structure

CRITICAL STRUCTURE RULE

* Claims MUST NOT be placed in incorrect conceptual sections

STRICT examples:

- Definitions MUST NOT appear under Applications
- Ethics MUST NOT appear under Types
- Concepts MUST NOT be mixed with consequences


SECTION DESIGN RULES (STRICT)

* Each section MUST represent ONE clear theme
* Section titles MUST be:
  - Specific
  - Multi-word
  - Derived from claim meaning

* Sections MUST NOT:
  - Mix unrelated claims
  - Mix opposing categories (when separable)

* If a claim contains mixed meaning:
  → Assign to MOST dominant theme
  → DO NOT split claim

ORDERING RULE (IMPORTANT)

Within each section, maintain logical order:

* Definition → explanation → implication
OR
* General → specific
